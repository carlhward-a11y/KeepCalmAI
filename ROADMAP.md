# KeepCalmAI Version 1 Roadmap

This roadmap is locked for Version 1. New ideas move to Version 2 unless they are needed to complete, fix or make Version 1 accessible.

## Version 1 goal

Ship a polished, accessible and confidence-building AI learning platform with two guided journeys.

## M1 Core Platform

Status: in progress.

Included:

- Dashboard
- Onboarding
- Learner Profile
- Journey Player
- Journey Map
- Progress Resume
- Achievements
- Prompt Workspace
- Notebook
- Recommendation foundation

Exit criteria:

- Progress-aware components use Learner Profile service
- Prompt Workspace is visible and usable
- Notebook displays saved learner artefacts
- Dashboard reflects learner progress

## M2 Learning Engine

Status: in progress.

Included:

- Learning DNA updates
- Adaptive feedback
- Prompt-building integration
- Reflection support
- Notebook integration
- Recommendation improvements

Exit criteria:

- Choices give meaningful feedback
- Prompt-building choices update the workspace
- Useful prompts can be saved
- Learning DNA changes based on learner activity

## M3 Journey Polish

Status: not started.

Included:

- AI Without Overwhelm final editorial pass
- Writing Better Prompts final editorial pass
- Mentor guidance review
- Choice feedback review
- Prompt flow review

Exit criteria:

- Every step has one clear idea
- Every interaction has a purpose
- Feedback is calm and helpful
- Both journeys can be completed without confusion

## M4 Documentation

Status: in progress.

Included:

- Product vision
- Architecture Handbook
- Developer Guide
- Journey Authoring Guide
- Component Catalogue
- Decision Log
- Release notes

Exit criteria:

- Documentation reflects the actual implementation
- Codex tasks have clear acceptance criteria
- Product philosophy is captured clearly

## M5 Accessibility and QA

Status: not started.

Included:

- Keyboard testing
- Screen reader testing
- Mobile testing
- Contrast review
- Reduced motion review
- Empty state review
- Error state review

Exit criteria:

- Both journeys work by keyboard
- Main flows are screen-reader sensible
- Mobile layout is usable
- Known issues are documented

## M6 Release Candidate

- Release checklist

- Release notes complete
- No known blocking issues
