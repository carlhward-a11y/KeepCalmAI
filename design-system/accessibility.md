# Design System Accessibility

Accessibility is part of the design system from the start.

It is not a final check or optional enhancement.

## Core standards

- Use clear structure and headings
- Support keyboard navigation
- Provide meaningful focus states
- Maintain strong colour contrast
- Avoid relying on colour alone
- Provide alt text for meaningful images
- Use plain language
- Keep cognitive load low
- Make progress clear
- Avoid time pressure where possible

## Cognitive accessibility

KeepCalmAI is designed for people who may feel overwhelmed.

Pages should use predictable layouts, short sections and one main action at a time.

## Screen reader access

Interactive elements must have clear names.

Headings should describe the page structure.

Decorative images should be hidden from assistive technology.

Meaningful images should have useful alt text.

## Magnification and zoom

Layouts should work well when enlarged.

Avoid fixed containers that break at high zoom.

## Motion

Animations should be optional where possible.

Do not use flashing or distracting movement.

## Inclusive design principle

If a design makes learning calmer and clearer for disabled users, it is likely to improve the experience for everyone.
